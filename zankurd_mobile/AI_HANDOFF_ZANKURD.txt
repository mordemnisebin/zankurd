Current Flutter App Behavior
----------------------------
On startup:
- If Supabase URL/key are provided, app initializes Supabase.
- If not, app runs in mock mode.
- App attempts to ensure anonymous profile.
- App loads categories from Supabase.
- App loads approved questions from Supabase.
- If Supabase fails, app falls back to local mock questions/categories.

Home screen:
- Shows ZanKurd branding.
- Shows actions:
  - Oda Kur
  - Kodla Katıl
  - Hızlı Yarış
  - Öğren
- Shows room preview.
- Shows first loaded question.
- Shows categories.

Oda Kur:
- Calls repository.createOnlineRoom().
- Supabase implementation:
  - anonymous sign-in if needed
  - upsert profile
  - find category ID
  - insert row into rooms
  - insert host into room_players
  - return GameRoom with Supabase room ID/code/player list
- If online creation fails, falls back to local mock room.

Kodla Katıl:
- Opens bottom sheet with room code.
- Calls repository.joinOnlineRoom(code).
- Supabase implementation:
  - anonymous sign-in if needed
  - upsert profile
  - find room by code
  - insert into room_players
  - tolerate duplicate join error 23505
  - return GameRoom with player list
- If join fails, falls back to local mock join.

Room screen:
- Shows room code, category, player list.
- Has "Hazırım" toggle.
- Has "Yarışı Başlat".
- Refreshes room players from Supabase every 4 seconds if room.id exists.
- Start game loads questions from Supabase and opens QuizScreen.

Quiz screen:
- Shows score/streak/progress.
- Shows current question and answers.
- Shows correct/wrong after answering.
- Local scoring:
  - correct answer gives 100 + streak bonus capped by clamp logic.
- Has bookmark and report UI stubs.
- Has 50/50 UI stub.
